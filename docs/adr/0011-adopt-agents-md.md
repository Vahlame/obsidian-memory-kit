# ADR-0011: `AGENTS.md` as canonical agent surface (IDE-agnostic)

- **Status:** Accepted
- **Date:** 2026-05-13
- **Deciders:** maintainer

## Context

v1 optimized for Cursor (`PROMPT_ULTRA_COMPLETO.md`, Windows paths). Other agents (Codex CLI, Copilot, Aider, Windsurf, Zed) increasingly read `AGENTS.md` and modular rule folders. Duplicated guidance across `MEMORY.md` (vault) vs repo root confused the split between “user vault memory” and “repo contributor instructions.”

## Decision

Make **`AGENTS.md` at the repository root** the single canonical document for agents working on this repo: overview, setup, style, tests, PRs, security, and the **memory protocol** for vault-backed workflows. Modular rules live under **`.agents/rules/`** and are synced into `.cursor/rules/*.mdc`, `.continue/rules/`, and the autogenerated block in `AGENTS.md` via `scripts/sync-agents.ts`. Symlinks (where the OS and `core.symlinks=true` allow) point `CLAUDE.md`, `.clinerules`, and `.github/copilot-instructions.md` at `AGENTS.md` for Claude Code, Cline, and Copilot.

## Consequences

- **Positive:** One source of truth; better multi-IDE discoverability; CI can enforce drift with `--check`.
- **Negative:** Windows contributors must enable symlink support or accept fallback copies documented in `docs/testing/manual-checks.md`.
- **Neutral:** The v1 ultra-prompt remains archived under `docs/legacy/` for historical installs.

## Alternatives considered

- **Cursor-only `.cursor/rules`:** Rejected — perpetuates vendor lock-in.
- **Keep `MEMORY.md` at repo root:** Rejected for this repo — no longer applicable; vault examples keep `examples/MEMORY.md` as a template only.
