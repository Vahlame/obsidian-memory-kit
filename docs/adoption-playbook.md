# Adoption Playbook (for teams)

## Objetivo

Adoptar memoria operativa compartida sin friccion.

## Fase 1 - Piloto (1 semana)

- 1-2 personas implementan setup completo.
- Validan estabilidad de MCP y sync.
- Ajustan plantilla de `PROJECTS/TEMPLATE.md`.

## Fase 2 - Estandarizacion (1-2 semanas)

- Definir convencion de naming para `PROJECTS/*.md`.
- Definir que va a `MEMORY.md` y que no.
- Publicar "prompt de cierre de tarea" interno.

## Fase 3 - Rollout equipo

- cada integrante instala con one-click;
- pega User Rules;
- corre `doctor.ps1`.

## Politica de calidad

- cada decision relevante: proyecto + fecha + motivo;
- cada cierre de tarea: 2-5 lineas en `SESSION_LOG.md`;
- no duplicar contenido entre archivos.

## Medidas de exito

- menos contexto repetido en chats;
- menor tiempo de onboarding;
- mayor consistencia entre decisiones.
